'runningScore' happens when a button is clicked, so we could call it then, from the three event handlers – but we’d have to call it three times.

Or, when the playerWins or computerWins procedures run – but then we still call it from two places.

Perhaps the best place to call it is from the findWinner procedure, at the bottom, outside the if…then block, so it runs every time and runs after the scores have been updated.

Add the ‘call runningScore’ block to the findWinner procedure.
